# Domain Documentation

## 原则

本仓库遵循 **单上下文领域文档** 规则：

1. **单一上下文**：每个 AI Agent 会话内，所有领域知识集中在一个 `docs/agents/` 目录中，不分散到多个位置。
2. **按职责拆分**：知识按关注点（Issue 追踪、标签分类、领域规则）分为独立文件，避免单一文件过于臃肿。
3. **显式引用**：AI Agent 在操作前必须读取相关领域文档，不可依赖内部知识。

## 文档清单

| 文件 | 用途 |
|------|------|
| `issue-tracker.md` | Issue 操作流程与绑定仓库 |
| `triage-labels.md` | 标签定义与使用规则 |
| `domain.md` | 本文档 — 领域文档的编写与组织规则 |

## 更新流程

- 新增领域知识时，优先在 `docs/agents/` 中新增文件或扩展现有文件
- 修改规则时同步更新 `AGENTS.md` 中的 skills 配置区块
- 每次提交前确保文档之间无矛盾
